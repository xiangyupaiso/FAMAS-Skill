# Workflow B: 行业/偏好筛选 (Screen by Preference)

## 工作流概述

根据用户给定的行业、主题或风格偏好，从全市场基金池中筛选匹配标的，并进行横向对比。

## 执行流程

```
用户输入: 行业/主题/风格偏好
    |
    v
+----------------------+
| sector_screener      | --> 全市场初筛(Top 50)
+-----------+----------+
            |
+-----------v-----------+
| fund_comparator      | --> 对Top 10进行横向PK
+-----------------------+
```

## 详细步骤

### Step 1: 需求解析与初筛 (sector_screener)

**输入**: 用户偏好描述

**执行**:
- 解析用户偏好类型（行业/主题/风格/资产类别）
- 确定筛选范围（主动/被动/ETF/联接）
- 全市场初筛，生成候选池(Top 50)
- 评估风格纯度
- 生成ETF联接映射

**输出**:
```json
{
  "user_preference": {
    "type": "偏好类型",
    "category": "具体类别"
  },
  "screening_logic": "筛选逻辑说明",
  "candidate_pool": {
    "count": 50,
    "funds": [
      {
        "fund_code": "代码",
        "fund_name": "名称",
        "match_score": "匹配度评分"
      }
    ]
  }
}
```

### Step 2: 横向对比 (fund_comparator)

**输入**: Top 10候选基金

**执行**:
- 收益风险对比
- 持仓重合度分析
- 成本对比
- 经理稳定性对比
- 生成淘汰/保留建议

**输出**:
```json
{
  "comparison": {
    "ranking": "综合排名",
    "eliminate": ["淘汰建议"],
    "retain": ["保留建议"]
  }
}
```

## 使用示例

```
用户: 我想找科技成长风格的基金

系统:
1. sector_screener 筛选科技成长型候选基金(Top 50)
2. fund_comparator 对Top 10进行横向PK

输出: Top 10候选基金 + 匹配度评分 + 筛选逻辑说明 + 淘汰/保留建议
```
