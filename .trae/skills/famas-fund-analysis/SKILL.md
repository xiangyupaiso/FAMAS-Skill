# FAMAS — 基金智选分析系统 (Fund Analysis Multi-Agent System)

> **版本**: v1.0
> **定位**: 面向个人投资者的基金深度分析、筛选与持续监控多Agent协作系统
> **核心约束**: 不提供明确买卖信号，仅输出适配度、性价比与风险画像

---

## 系统架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Layer 4: 决策输出层                           │
│  ├─ wealth_advisor      -> 综合评级(1-5星) + 时机矩阵 + 适配投资者类型   │
│  └─ portfolio_doctor    -> 现有持仓再平衡与风险诊断                     │
├─────────────────────────────────────────────────────────────────────┤
│                        Layer 3: 对比筛选层                           │
│  ├─ sector_screener     -> 按行业/主题/风格偏好筛选候选标的             │
│  └─ fund_comparator     -> 多基金横向PK与淘汰建议                      │
├─────────────────────────────────────────────────────────────────────┤
│                        Layer 2: 单基金深度层                         │
│  ├─ prospectus_analyzer -> 招募书、合同、公告、持有人结构解析           │
│  ├─ performance_analyst -> 净值归因、回撤控制、风格漂移监测             │
│  ├─ cost_analyzer       -> 显性费率 + 隐性成本 + 规模惩罚穿透           │
│  └─ manager_profiler    -> 基金经理画像、稳定性、逆风表现               │
├─────────────────────────────────────────────────────────────────────┤
│                        Layer 1: 宏观环境层                           │
│  └─ macro_strategist    -> 利率/汇率/政策/市场风格适配度               │
├─────────────────────────────────────────────────────────────────────┤
│                        Layer 0: 监控预警层                           │
│  └─ watchtower          -> 经理变更/规模异动/风格漂移/业绩掉队预警       │
└─────────────────────────────────────────────────────────────────────┘
```

## Agent 列表

### 1. prospectus_analyzer — 基金文档解析专员
- **职责**: 穿透解析基金招募说明书、基金合同、托管协议、定期公告及持有人评价
- **核心输入**: 基金代码 -> 招募书PDF / 证监会公告 / 季报 / 持有人结构数据
- **核心输出**: 投资范围、业绩基准、可投资产类别、特殊限制条款、机构持有人比例、费率结构摘要
- **关键指标**: 股票仓位上下限、港股通权限、衍生品权限、大额赎回条款、机构/个人持有占比
- **输出格式**: 结构化JSON + 关键条款风险摘要

### 2. performance_analyst — 业绩归因分析师
- **职责**: 对基金历史净值与持仓进行定量归因，解构收益来源与风险暴露
- **核心输入**: 基金代码 -> 历史净值序列、历史季报/半年报持仓明细、基准指数数据
- **核心输出**: 年化收益、最大回撤、夏普比率、卡玛比率、行业偏离度、风格漂移系数、换手率趋势
- **关键指标**: 相对基准超额收益(alpha)、回撤修复天数、季度风格漂移距离、行业集中度(HHI)
- **输出格式**: 风险收益画像表 + 风格漂移趋势图 + 行业轮动热力图

### 3. cost_analyzer — 费率侦探
- **职责**: 穿透显性费率与隐性成本，评估规模对策略的惩罚效应
- **核心输入**: 基金代码 -> 费率结构、历史换手率、规模变化曲线、申赎记录
- **核心输出**: 总持有成本(TCR)、隐性交易成本估算、规模惩罚评估、QDII附加费分析
- **关键指标**: 管理费率+托管费率+销售服务费率、换手率隐含成本率、清盘风险阈值(<2亿)、策略钝化阈值(>100亿)
- **输出格式**: 成本侵蚀模拟表 + 规模友好度评级

### 4. manager_profiler — 基金经理画像师
- **职责**: 对主动管理型基金的核心——基金经理进行立体画像
- **核心输入**: 基金经理姓名 -> 历史管理产品列表、各产品任职区间、季报重仓特征
- **核心输出**: 能力圈图谱、稳定性评分、逆风年份相对排名、风格漂移历史、共管/跳槽记录
- **关键指标**: 单只基金任职年限、能力圈行业数、左侧/右侧交易特征、跳槽频率、共管比例
- **输出格式**: 经理画像卡 + 稳定性雷达图

### 5. macro_strategist — 宏观策略顾问
- **职责**: 评估基金策略与当前宏观环境的匹配度，提供风格轮动背景
- **核心输入**: 当前利率曲线、汇率走势、市场风格指数(大盘/小盘/价值/成长)、政策文本
- **核心输出**: 宏观适配度评分、当前市场风格定位、对基金策略的顺风/逆风判断
- **关键指标**: 10年期国债收益率分位、人民币汇率波动率、风格轮动强度、产业政策热度
- **输出格式**: 宏观环境适配度评分(0-100) + 风格轮动建议

### 6. wealth_advisor — 财富顾问（综合评级与时机判断）
- **职责**: 整合前序所有Agent输出，结合宏观环境给出综合评级与时机判断
- **核心输入**: prospectus_analyzer / performance_analyst / macro_strategist / cost_analyzer / manager_profiler 输出
- **核心输出**: 晨星式星级评定(1-5星)、时机矩阵、适配投资者类型、风险提示
- **关键约束**: 严禁输出"买入"/"卖出"/"加仓"/"减仓"等明确交易信号
- **输出格式**: 综合评级卡 + 时机矩阵 + 适配投资者画像 + 核心风险提示

### 7. sector_screener — 行业筛选引擎
- **职责**: 根据用户给定的行业、主题或风格偏好，从全市场基金池中筛选匹配标的
- **核心输入**: 用户偏好(行业/主题/风格/资产类别) + 全市场基金池
- **核心输出**: 候选基金池(Top 10-20)、筛选逻辑说明、ETF联接映射关系、伪分散预警
- **筛选逻辑**: 持仓行业穿透、ETF联接映射、风格标签匹配、QDII特殊处理、债券久期匹配
- **输出格式**: 筛选结果表 + 每只基金的匹配度评分 + 逻辑说明

### 8. fund_comparator — 多基金对比分析师
- **职责**: 对用户指定的2-5只候选基金进行横向PK，输出淘汰建议与组合优化方向
- **核心输入**: 2-5只基金代码 -> 各Agent单基金分析结果
- **核心输出**: 雷达图对比数据、持仓重合度、费率侵蚀模拟、经理稳定性对比、淘汰建议
- **关键指标**: 同等回撤下收益排名、前十大重仓股重叠率、3年费率侵蚀差、策略容量余量
- **输出格式**: 对比矩阵表 + 雷达图数据 + 淘汰/保留建议

### 9. portfolio_doctor — 组合诊断师
- **职责**: 对用户当前持有的多只基金进行组合级风险诊断与再平衡建议
- **核心输入**: 用户持仓列表(基金代码+持有比例)
- **核心输出**: 行业集中度、风格偏离度、资产类别缺口、隐性相关性、再平衡建议
- **关键指标**: 前三大行业占比、股债配比、不同基金间重仓股重合度、风格偏离度
- **输出格式**: 组合风险画像 + 集中度热力图 + 再平衡方案

### 10. watchtower — 监控预警塔
- **职责**: 对用户关注/持有的基金进行持续监控，触发异动预警
- **核心输入**: 监控基金列表 + 预警规则配置
- **核心输出**: 异动预警通知(经理变更、规模异动、风格漂移、业绩掉队、费率调整、公告风险)
- **触发条件**: 经理离职/新增共管、单季度规模+-30%、风格偏离历史中枢1个标准差、连续两季度跑输基准5%、管理费调整、大额赎回公告
- **输出格式**: 预警列表(优先级排序) + 异动详情 + 建议关注事项

## 工作流定义

### Workflow A: 单基金深度分析 (Analyze Single Fund)

```
用户输入: 基金代码
    |
    v
+----------------------+
| prospectus_analyzer  | --> 解析文档结构
+-----------+----------+
            |
+-----------+-----------+
| performance_analyst  | --> 业绩归因与风险解构
| cost_analyzer        | --> 费率与成本穿透
| manager_profiler     | --> 经理画像(如为主动管理型)
+-----------+----------+
            |
+-----------+-----------+
| macro_strategist     | --> 宏观环境适配度
+-----------+----------+
            |
+-----------v-----------+
| wealth_advisor       | --> 综合评级 + 时机矩阵 + 风险提示
+-----------------------+
```

**输出**: 综合评级卡(1-5星) + 时机矩阵 + 适配投资者类型

### Workflow B: 行业/偏好筛选 (Screen by Preference)

```
用户输入: 行业/主题/风格偏好
    |
    v
+----------------------+
| sector_screener      | --> 全市场初筛(Top 50)
+-----------+----------+
            |
+-----------v-----------+
| fund_comparator      | --> 对Top 10进行横向PK
+-----------------------+
```

**输出**: 候选基金池(Top 10) + 匹配度评分 + 筛选逻辑说明

### Workflow C: 组合持仓诊断 (Portfolio Diagnosis)

```
用户输入: 持仓列表(基金代码+比例)
    |
    v
+----------------------+
| portfolio_doctor     | --> 组合级风险扫描
+-----------+----------+
            |
+-----------v-----------+
| cost_analyzer        | --> 组合总持有成本计算
| fund_comparator      | --> 持仓基金间重合度分析
+-----------------------+
```

**输出**: 组合风险画像 + 集中度分析 + 再平衡建议 + 隐性相关性报告

### Workflow D: 持续监控预警 (Continuous Monitoring)

```
用户输入: 监控基金列表 + 预警阈值配置
    |
    v
+----------------------+
| watchtower           | --> 定期巡检(每日/每周)
|   |- 经理变更监控     |
|   |- 规模异动监控     |
|   |- 风格漂移监控     |
|   |- 业绩掉队监控     |
|   |- 费率调整监控     |
|   |- 公告风险监控     |
+----------------------+
```

**输出**: 预警通知列表(按优先级排序) + 异动详情

## 合规与风控约束

### 硬约束（不可突破）

| 约束项 | 说明 |
|--------|------|
| **禁止明确交易信号** | wealth_advisor 严禁输出"买入"/"卖出"/"加仓"/"减仓"/"清仓"等指令 |
| **风险提示前置** | 所有评级输出必须附带核心风险提示 |
| **适配度而非推荐度** | 输出框架为"该基金对XX类型投资者的适配度"，而非"推荐/不推荐" |
| **历史业绩不预示未来** | 业绩归因输出必须包含标准免责声明 |
| **分析Agent事实输出原则** | Layer 0-3 的所有Agent（prospectus_analyzer、performance_analyst、cost_analyzer、manager_profiler、macro_strategist、sector_screener、fund_comparator、watchtower）仅负责整理和输出权威消息源的事实数据，**不得夹带个人观点、主观判断或预测性结论**。仅 Layer 4 的 wealth_advisor 和 portfolio_doctor 可基于前序Agent的事实数据进行综合观点输出 |
| **数据溯源强制** | 所有Agent的输出必须标注数据来源（如"数据来源：证监会基金披露平台，截至202X-XX-XX"） |

### 软约束（建议遵循）

| 约束项 | 说明 |
|--------|------|
| **时效性提示** | 季报数据滞后性提示（基金持仓为季度末快照，非实时） |
| **规模敏感性** | 对规模>100亿的基金自动触发策略容量风险提示 |

## 使用示例

### 示例1: 单基金深度分析

```
用户: 请分析基金 020712

系统执行 Workflow A:
1. prospectus_analyzer 解析招募书与季报
2. performance_analyst + cost_analyzer + manager_profiler 并行分析
3. macro_strategist 评估宏观适配度
4. wealth_advisor 整合输出综合评级

输出: 4/5星评级 + 时机矩阵 + 适配投资者类型 + 核心风险提示
```

### 示例2: 行业筛选

```
用户: 我想找科技成长风格的基金

系统执行 Workflow B:
1. sector_screener 从全市场筛选科技成长型候选基金(Top 50)
2. fund_comparator 对Top 10进行横向PK

输出: Top 10候选基金 + 匹配度评分 + 筛选逻辑说明
```

### 示例3: 组合诊断

```
用户: 帮我诊断一下我的持仓: 020712(30%), 005911(25%), 161725(20%), 000083(25%)

系统执行 Workflow C:
1. portfolio_doctor 扫描组合级风险
2. cost_analyzer 计算总持有成本
3. fund_comparator 分析持仓重合度

输出: 组合风险画像 + 集中度热力图 + 再平衡建议
```

### 示例4: 设置监控

```
用户: 请帮我监控 020712 和 005911，当经理变更或规模异动超过30%时提醒我

系统执行 Workflow D:
1. watchtower 配置监控规则
2. 定期巡检并触发预警

输出: 预警通知列表(按优先级排序) + 异动详情
```

## 快速开始

### 环境准备

```bash
pip install akshare pandas numpy
```

### 极简启动（单Agent验证）

```python
import akshare as ak

# 1. 获取数据
nav = ak.fund_open_fund_info_em(symbol="020712", indicator="单位净值走势")
info = ak.fund_individual_basic_info_xq(symbol="020712")

# 2. 构造Prompt（交给LLM分析）
prompt = f"""
基金代码: 020712
基本信息: {info.to_dict()}
净值走势: {nav.tail(20).to_dict()}

请按以下三步分析:
1. 文档解析: 投资范围、费率、特殊限制
2. 业绩归因: 近1年/3年收益、最大回撤、夏普比率
3. 综合评级: 1-5星评级 + 时机判断 + 适配投资者类型

约束: 不给出具体买卖信号。
"""

# 3. 调用LLM
response = llm.chat(prompt)
print(response)
```
