# Workflow D: 持续监控预警 (Continuous Monitoring)

## 工作流概述

对用户关注/持有的基金进行持续监控，当触发预设条件时输出预警通知。

## 执行流程

```
用户输入: 监控基金列表 + 预警阈值配置
    |
    v
+----------------------+
| watchtower           | --> 定期巡检(每日/每周)
|   |- 经理变更监控     |
|   |- 规模异动监控     |
|   |- 风格漂移监控     |
|   |- 业绩掉队监控     |
|   |- 费率调整监控     |
|   |- 公告风险监控     |
+----------------------+
```

## 详细步骤

### Step 1: 配置监控规则

**输入**: 监控基金列表 + 预警阈值配置

**默认监控规则**:

| 监控维度 | 触发条件 | 优先级 |
|----------|----------|--------|
| 经理变更 | 经理离职/新增共管 | 高 |
| 规模异动 | 单季度规模+-30% | 高 |
| 风格漂移 | 偏离历史中枢1个标准差 | 中 |
| 业绩掉队 | 连续两季度跑输基准5% | 高 |
| 费率调整 | 管理费调整 | 中 |
| 公告风险 | 大额赎回公告 | 高 |

**自定义配置**:
```json
{
  "watchlist": ["020712", "005911"],
  "alert_rules": {
    "manager_change": true,
    "scale_change_threshold": 0.30,
    "style_drift_threshold": 1.0,
    "performance_lag_quarters": 2,
    "performance_lag_threshold": 0.05,
    "fee_change": true,
    "announcement_risk": true
  },
  "check_interval": "daily"
}
```

### Step 2: 定期巡检

**执行**:
- 按配置间隔执行巡检
- 检查每只基金的各维度数据
- 判断是否触发预警条件
- 生成预警列表

**输出**:
```json
{
  "monitoring_date": "监控日期",
  "alerts": [
    {
      "fund_code": "基金代码",
      "alert_type": "预警类型",
      "priority": "优先级",
      "trigger_condition": "触发条件",
      "details": "异动详情"
    }
  ]
}
```

### Step 3: 预警通知

**输出格式**:

```markdown
## 基金监控预警报告 - {日期}

### 监控概览
- 监控基金数: {数量}
- 预警总数: {数量}
- 高优先级: {数量}
- 中优先级: {数量}
- 低优先级: {数量}

### 高优先级预警

#### {基金代码} - {基金名称}
- **预警类型**: {类型}
- **触发条件**: {条件}
- **异动详情**: {详情}
- **建议关注**: {建议}

### 中优先级预警
...

### 低优先级预警
...

---
*免责声明：本报告仅作信息整理与适配度分析，不构成投资建议。*
```

## 使用示例

```
用户: 请帮我监控 020712 和 005911，当经理变更或规模异动超过30%时提醒我

系统:
1. watchtower 配置监控规则
2. 每日/每周执行巡检
3. 触发条件时输出预警

输出: 预警通知列表(按优先级排序) + 异动详情
```

## 触发条件详解

### 经理变更
- 基金经理离职
- 新增共管基金经理
- 共管关系解除
- 经理任职状态变更

### 规模异动
- 单季度规模增长超过30%
- 单季度规模下降超过30%
- 规模突破100亿（策略容量风险）
- 规模跌破2亿（清盘风险）

### 风格漂移
- 风格偏离历史中枢1个标准差
- 季度风格漂移距离异常增大
- 持仓风格发生显著变化

### 业绩掉队
- 连续两季度跑输基准5%
- 同类排名下滑至后50%
- 超额收益持续为负

### 费率调整
- 管理费率上调
- 托管费率调整
- 新增费用项目

### 公告风险
- 大额赎回公告
- 清盘风险提示
- 其他重大不利公告
